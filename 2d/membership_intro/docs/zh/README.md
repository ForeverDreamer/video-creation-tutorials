# 频道介绍展示场景 — 免费配套包

频道介绍视频里用脚本生成的两个场景：一个会动的照片「英雄」卡，一片脉冲发光网格。纯 After Effects —— 不用插件、不用付费模板、无需安装任何东西。下载即跑，改 `CONFIG` 再跑一次。

## 包里有什么

| 文件 | 是什么 |
|------|--------|
| `code/demo_hero_scene.jsx` | 会动的照片英雄场景：圆角照片卡 + 发光边框 + 粗体标题 + 点缀环 + 漂浮点 + 单节点 3D 相机飞掠 —— 元素交错弹入，之后保持轻微持续微动。**素材可选**：指向你自己的照片，或留空用占位卡片。 |
| `code/demo_grid_pulse.jsx` | 一片圆角发光单元的对角行波网格，用 Opacity + Scale 上的正弦表达式让整片活起来。**零素材** —— 网格完全程序化生成。 |
| `code/assets/SOURCES.md` | 那张可选照片的来源台账。照片本体**不打包** —— 自己下（链接见下）。 |

## 英雄场景里的那张照片（可选）

英雄 demo **零素材**也能跑 —— 没图就用占位卡片。想用真照片：**把任意 `.jpg`/`.png` 放进脚本同目录的 `code/assets/` 文件夹**，自动加载 —— 无需填路径。想跟视频一样，它用的是一张免费 Pexels 猫照片：

- **猫照片** —— Pexels，免费、可商用、免署名：
  <https://www.pexels.com/photo/cat-lying-on-the-sofa-19511759/>

下载到 `code/assets/` 即可。我们不打包图片 —— 记下来源、自己下（见 `code/assets/SOURCES.md`）。想用显式路径？在脚本 `CONFIG` 里把 `IMAGE_PATH` 设成完整路径、用**正斜杠**（如 `"D:/photos/cat.jpg"`，别用反斜杠），会覆盖自动识别。

## 跑一下 demo

1. 打开 After Effects。
2. `文件 > 脚本 > 运行脚本文件...`，选 `code/` 下的某个 demo。
3. **英雄场景** → 建出带照片卡 + 相机飞掠的合成；按小键盘 `0` 做 RAM 预览。**网格脉冲** → 在当前合成（或新建一个）里铺满会动的发光网格；按 `0` 预览。

## 调成你自己的

所有可调项都在每个脚本顶部的 `CONFIG` 块里。

**`demo_hero_scene.jsx`**

- `IMAGE_PATH` — 留 `""` 自动用 `code/assets/` 里的图（放一张进去即可）；或填显式路径如 `"D:/photos/cat.jpg"`（正斜杠）覆盖。
- `TITLE_TEXT` — 照片下方的粗体标题。
- `PAL` — 配色板（`bg` / `card` / `gold` / `teal` / `violet` / `coral` / `title`），每个 `[r, g, b]`（0–1）。
- `CAM_KEYS` — 烤好的单节点相机飞掠；设成 `[]` 用静态默认相机。

**`demo_grid_pulse.jsx`**

- `COLS` — 网格列数（行数按画面自动推导铺满）。
- `GAP` — 单元间距，px。
- `PALETTE` — 青绿发光配色，每个 `[r, g, b]`（0–1）。

改一个值，再跑一次，整个场景立刻更新。

## 一个值得知道的点

- **英雄场景**用的是**单节点相机**（兴趣点 point-of-interest 为 null），靠 `Orientation` / `Rotate Z` 控朝向。别之后给它加兴趣点 —— 单节点相机加了兴趣点可能静默出错，或跟你的关键帧打架。保持单节点，跟脚本里一样。
- **网格脉冲**的「活」来自**表达式**（Opacity + Scale 上的正弦），不是关键帧 —— 所以它会一直循环，改合成时长也会自动重新对齐。单元是平面 2D 形状层，运动纯程序化。

## 关于这份免费代码

这是视频里演示的版本。直接跑可能需要小调整 —— After Effects 版本、字体、操作系统差异都可能不同。如果你只想**看懂这个场景怎么搭的**，看代码跑一下就够了。
